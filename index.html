HTML

  • Checkout MDN Web Docs for more information about HTML, CSS, and JavaScript!
  • HTML stands for Hypertext Markup Language.
  • The (head) element contians information about the webpage, or metadata. .
  • The (body) element represents the visible content shown to the user.
  • The (img) element contains information about imagines on the page, and the src tag defines the location of the image.
  • The (br) element creates a break, or a line of empty space on the page.
  • The (a) element stands for anchor and creates links to the same webpage or other webpages.
  • The (href) attribute points the anchor to a URL for the link.
  • The (section) element is located in the body, they are semantic and contian thematic content. Thematic content represents chunks of narrative or story-like content.
  • The (id) attribute is a way to identify elements and select them for use is CSS and JavaScript.

CSS

  • CSS stands for Cascading Style Sheets and is how we add style to a page.
  • The three types of CSS are: inline CSS, internal CSS, and external CSS.
  • Seperation of Concerns refers to seperating code into different files based on use. It is based on the idea that each section of code should have its own responsibility.
  • Work performed on CSS and JS happened inside the assets folder. Files were created inside the folder using the command line (touch) command.
  • Declarations contain two components: the CSS property we want to apply and its value.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Use the (link) element with the (rel) and (href) attributes in the head section to add CSS pages.
  • (Card) is a class selector and any element assigned to that class will gain it's properties.
  • Color can be assigned with either the name of the color or with a hex number from the HTML color codes site.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: add our changed files to the staging area of Git.
  • git commit -m "note about changes": commits the changes.
  • git pull origin main: makes sure that our current branch is up to date.
  • git push origin feature/branch-name: pushes the file up.
  • When executing a PR (pull request) make sure the base branch is main and the comparison branch is the feature/branch-name.
  • After creating a PR, merge the comparison branch into the base branch with the Merge PR button.
  • Close any open issuses reguarding the PR.
  • git check out main: returns you to the main branch.
  • After pushing an update, make sure your local main branch is up to date. (git checkout main followed by git pull origin main)

JavaScript

  • The script.js file is placed between the footer and the closing (body) element.
  • A variable is a name container that allows us to store data in our code.
  • An example of a variable is the (const) variable, which is a variable that can't be reassigned a value.
  • The (console.log) method followed by an argument, which was topics in the study guide, will cause an output on the web console.
  • Control flow is the order in which a computer executes code in a script.
  • Control flow can be interuped with things like conditional statements and loops.
  • Conditional statements (if) cause the computer to execute the action in curvy brackets if the statement is truthy.
  • The strict equality operator (===) checks to see if two values are equal, and returns a Boolean result true if the values are equal and false if the values are not equal.
  • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.
  • A (for) loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Arrays are useful tools for storing multiple items with different data types in a single variable, but not all arrays must have multiple data types.
  • In JavaScript and many other programming languages, arrays are zero-indexed and sequential. This means that the first item in any array has an index of 0, not 1. The indices, or location identifiers, will always be whole numbers that increase by 1 each time.
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
  • A function's name is followed by parentheses (). The consle.log method will log the string in () after it is called.
  • To call a function, we use the name of the function in our JavaScript file where we want the code to execute.
